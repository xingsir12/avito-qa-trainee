## 1. POST /api/1/item — Создать объявление

### Тело запроса (схема)
```json
{
  "sellerID": <integer>,
  "name": <string>,
  "price": <integer>,
  "statistics": {
    "likes": <integer>,
    "viewCount": <integer>,
    "contacts": <integer>
  }
}
```

### Ожидаемый ответ 200
```json
{
  "id": "<string>",
  "sellerId": "<integer>",
  "name": "<string>",
  "price": "<integer>",
  "statistics": { "likes": ..., "viewCount": ..., "contacts": ... },
  "createdAt": "<string>"
}
```

---

### TC-1.1 [Позитивный] Создание объявления с валидными данными

**Предусловие:** сервис доступен

**Шаги:**
1. Отправить POST `/api/1/item` с телом:
```json
{
  "sellerID": 123456,
  "name": "Test Item",
  "price": 1000,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 1 }
}
```

**Ожидаемый результат:**
- Статус: `200 OK`

## 7. E2E сценарии

---

### TC-7.1 Полный жизненный цикл объявления

**Шаги:**
1. POST `/api/1/item` → сохранить `id`
2. GET `/api/1/item/{id}` → данные совпадают с переданными
3. GET `/api/1/statistic/{id}` → статистика корректна
4. DELETE `/api/2/item/{id}` → `200 OK`
5. GET `/api/1/item/{id}` → `404 Not Found`

**Ожидаемый результат:** все шаги прошли без ошибок

---

### TC-7.2 Несколько объявлений одного продавца: создать → удалить одно → проверить список

**Шаги:**
1. POST × 3 с одним `sellerID` → сохранить все `id`
2. GET `/api/1/{sellerID}/item` → все 3 наших объявления присутствуют
3. DELETE `/api/2/item/{id[0]}`
4. GET `/api/1/{sellerID}/item` → id[0] отсутствует, остальные два присутствуют

**Ожидаемый результат:** список корректно обновляется после удаления

---

### TC-7.3 Согласованность данных между эндпоинтами

**Шаги:**
1. POST → сохранить `id`, `sellerID`, `name`, `price`, `statistics`
2. GET `/api/1/item/{id}` → проверить совпадение полей
3. GET `/api/1/{sellerID}/item` → найти объявление в списке, проверить поля
4. GET `/api/1/statistic/{id}` → проверить статистику

**Ожидаемый результат:** данные одинаковы во всех трёх ответах

---

### TC-7.4 Валидация формата createdAt

**Шаги:**
1. Создать объявление
2. Проверить поле createdAt в ответе

**Ожидаемый результат:**
- Формат ISO 8601: `2024-01-15T10:30:00.000Z`
- Время серверное (не локальное)

---

### TC-7.5 Проверка сортировки в списке продавца

**Шаги:**
1. Создать 3 объявления с одним sellerId с интервалом 1 секунда
2. GET `/api/1/{sellerID}/item`

**Ожидаемый результат:**
- Объявления отсортированы по убыванию createdAt (новые первые)
- ИЛИ документация описывает порядок сортировки
